# Plan 1 — Estructura base y layout del portfolio

## Prompt a la IA
"Necesito planificar la estructura y arquitectura de una plantilla profesional de portfolio web usando HTML5, CSS3 y JavaScript vanilla, siguiendo buenas prácticas de diseño responsive, semántica HTML y organización de archivos."

## Respuesta de la IA (Plan)

### Objetivo del sprint
Construir una base sólida y reutilizable de portfolio, con estructura semántica, arquitectura CSS clara y despliegue funcional en GitHub Pages.

### Alcance
- No incluir aún animaciones complejas
- No incluir contenido definitivo
- Priorizar estructura, responsive y orden

### Estructura del proyecto
- Usar un archivo index.css como punto de entrada
- JavaScript mínimo para interacciones básicas

### Estructura HTML
- Uso de etiquetas semánticas:
  - header con navegación
  - main con secciones: hero, about, projects, contact
  - footer con información legal y enlaces
- Jerarquía correcta de encabezados (un solo h1)

### Responsive
- Enfoque mobile-first
- Tipografía fluida usando clamp()
- Layout con Flexbox y CSS Grid
- Sin scroll horizontal en móvil

### Accesibilidad básica
- Buen contraste de color
- Texto alternativo en imágenes
- Navegación funcional con teclado

### Despliegue
- Repositorio en GitHub
- GitHub Pages configurado
- Página 404 personalizada

## Implementación real
- Se implementó la estructura de carpetas
- Se creó index.html semántico
- Se configuró base.css con reset y variables
- Se desplegó correctamente en GitHub Pages
